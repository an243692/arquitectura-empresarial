# Arquitectura Empresarial

## Descripcion
Repositorio de practica para la implementacion de conceptos de **Control de Versiones** y **Arquitectura de Software Empresarial**.

## Estructura Trunk-Based
- **Trunk (main)**: Linea de desarrollo principal y estable
- - **Branches**: Ramas de caracteristicas para desarrollo paralelo
  - - **Tags/Releases**: Versiones marcadas del software empresarial
   
    - ## API REST
    - Este proyecto documenta una API REST:
    - - **Interoperabilidad**: Basada en estandares HTTP
      - - **Rendimiento**: Formato JSON ligero
        - - **Escalabilidad**: Arquitectura sin estado
         
          - ## Comandos Git del Ciclo de Trabajo
          - \```bash
          - git clone <url>       # Copia de trabajo local
          - git pull              # Actualizar copia
          - git add .             # Preparar cambios
          - git commit -m "msg"   # Registrar cambios
          - git push              # Enviar al repositorio remoto
          - \```
